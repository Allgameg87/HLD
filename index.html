1.4 Назначение документа

Данный документ High Level Design (HLD) описывает высокоуровневую архитектуру системы "Умный дом SmartHome Pro". Цель документа - предоставить общее понимание системы, ее компонентов и взаимодействия между ними для всех заинтересованных сторон.

Документ предназначен для:

  • Команды разработчиков - как руководство для реализации
  • Менеджеров проекта - для понимания масштаба и сложности системы
  • Заказчиков - для понимания возможностей системы
  • Технических специалистов заказчика - для планирования развертывания

Документ не содержит низкоуровневых технических деталей реализации, которые будут описаны в соответствующих LLD (Low Level Design) документах.

2. ТЕХНИЧЕСКАЯ ПОСТАНОВКА ЗАДАЧИ

Система "Умный дом SmartHome Pro" предназначена для автоматизации управления инженерными системами жилых и коммерческих помещений. Основные задачи системы:

  • Централизованное управление освещением, климатом, безопасностью и мультимедиа
  • Автоматизация рутинных операций через сценарии
  • Мониторинг состояния всех подключенных устройств
  • Обеспечение удаленного доступа к управлению через мобильные устройства
  • Интеграция с популярными сервисами (голосовые помощники, погодные сервисы и др.)
  • Обеспечение энергоэффективности за счет оптимизации работы устройств

Система должна быть:

  • Масштабируемой - возможность добавления новых устройств и функций
  • Надежной - с минимальным временем простоя
  • Безопасной - с защитой от несанкционированного доступа
  • Простой в установке и использовании
  • Совместимой с популярными стандартами умного дома

Основные требования к системе:

  • Поддержка не менее 50 устройств в одной системе
  • Время отклика на локальные команды - не более 1 секунды
  • Автономная работа локальных сценариев при отсутствии интернета
  • Хранение истории событий не менее 30 дней
  • Поддержка голосового управления через популярные платформы

3. ОПИСАНИЕ ТЕХНИЧЕСКОГО РЕШЕНИЯ

Система "Умный дом SmartHome Pro" построена по модульному принципу с центральным управляющим устройством (HUB) и набором подключаемых устройств. Архитектура системы включает:

  • Центральный HUB - выполняет функции управления, хранения данных и обеспечения связи между компонентами
  • Устройства управления - датчики, выключатели, контроллеры
  • Исполнительные устройства - реле, диммеры, климатическая техника
  • Облачный сервер - для удаленного доступа, хранения истории и интеграций
  • Мобильное приложение - интерфейс управления для пользователей

Система поддерживает несколько протоколов связи для работы с различными устройствами:

  • Zigbee - для датчиков и исполнительных устройств
  • Wi-Fi - для устройств с высоким энергопотреблением
  • Bluetooth LE - для персональных устройств

3.2 Системные требования для установки

Требования к оборудованию:

  • Центральный HUB:
    • Процессор: 4-ядерный ARM Cortex-A53 1.2 ГГц
    • Память: 1 ГБ RAM, 8 ГБ Flash
    • Интерфейсы: Ethernet, Wi-Fi 802.11ac, Zigbee 3.0, Bluetooth 5.0
    • Питание: 12V DC или PoE
  • Сетевая инфраструктура:
    • Шлюз в интернет с пропускной способностью не менее 10 Мбит/с
    • Локальная сеть (рекомендуется гигабитная)
    • Резервное питание для HUB (рекомендуется)

Требования к мобильным устройствам для управления:

  • Android 8.0 и выше или iOS 12 и выше
  • Подключение к интернету
  • Рекомендуется экран с диагональю не менее 4.7"

3.3 Схема включения и описание схемы

Архитектурная схема системы "Умный дом SmartHome Pro"
+-----------------------------------+ | | | Облачный сервер | | (Удаленное управление, | | история событий, API) | | | +------------------+----------------+ | | HTTPS/API +------------------+----------------+ | | | Центральный HUB | | (Логика управления, локальное | | хранение, интеграция устройств) | | | +--------+-------------------+---------------+--------+ | | | | Zigbee | Wi-Fi | +----------------------------+------------------------+ | | | | Zigbee-устройства: | Wi-Fi устройства: | | - Датчики | - Камеры | | - Умные выключатели | - Умные розетки | | - Диммеры | - Медиаустройства | | | | +----------------------------+------------------------+

Система построена по трехуровневой архитектуре:

  1. Уровень устройств - включает все датчики и исполнительные устройства, подключенные по Zigbee, Wi-Fi или Bluetooth
  2. Уровень управления (HUB) - центральное устройство, которое:
    • Обеспечивает связь между устройствами
    • Выполняет локальные сценарии
    • Хранит текущее состояние системы
  3. Облачный уровень - обеспечивает:
    • Удаленный доступ через мобильные приложения
    • Хранение истории событий
    • Интеграцию с внешними сервисами

Типовая схема подключения устройств в квартире:

  1. HUB устанавливается в центре помещения и подключается к роутеру
  2. Zigbee-устройства подключаются непосредственно к HUB
  3. Wi-Fi устройства подключаются через локальную сеть
  4. Для расширения зоны покрытия Zigbee могут использоваться ретрансляторы

3.4 Описание системы резервного копирования

Система предусматривает несколько уровней резервирования:

  • Локальное хранение конфигурации - все настройки системы хранятся во внутренней памяти HUB с возможностью экспорта в файл
  • Облачное резервное копирование - конфигурация автоматически синхронизируется с облаком при изменениях
  • Резервирование сценариев - пользовательские сценарии могут быть экспортированы отдельно

Процедура восстановления:

  1. Загрузка последней резервной копии из облака или локального файла
  2. Поэтапное подтверждение восстановления параметров
  3. Автоматическое переподключение устройств

Заказчик обязуется:

  • Обеспечить доступ к помещениям для монтажа оборудования в согласованные сроки
  • Предоставить информацию о специфике эксплуатации помещений
  • Обеспечить доступ к электросети и интернету в местах установки оборудования
  • Назначить ответственных лиц для взаимодействия с исполнителем
  • Обеспечить соблюдение условий эксплуатации оборудования
  • Своевременно информировать о неисправностях системы

6.2 Соглашение об уровне обслуживания (SLA)

Гарантии:

  • Гарантия на оборудование - 2 года
  • Гарантия на работы по монтажу - 1 год
  • Доступность облачного сервиса - 99.5% в год

Время реакции на инциденты:

Критичность Время реакции Время устранения
Критическая (полная неработоспособность) 1 час 24 часа
Высокая (частичная неработоспособность) 4 часа 3 рабочих дня
Средняя (ограниченная функциональность) 8 часов 5 рабочих дней
